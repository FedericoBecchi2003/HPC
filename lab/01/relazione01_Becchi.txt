Ora si asseggnano i permessi a /root
chmod 755 /root
così root può scrivere e altri possono leggere/entrare

Creazione chiavi SSH
creazione chiavi con algoritmo RSA con nessuna passphrase e viene salvata dentro ~/.ssh/id_rsa
ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa
in output si ottengo due file:
id_rsa → chiave privata
id_rsa.pub → chiave pubblica

abilitazione accesso automatico
cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keyss

protezione alla cartella delle chiavi
chmod 600 ~/.ssh/authorized_keys
solo il propietario può leggere e scrivere
gli altri user l'acceso è proibito

wn1 si autentica su wn2 tramite una chiave crittografica RSA invece che con password, permettendo comunicazione automatica tra nodi (fondamentale per MPI).

3) Determinare le prestazioni di picco in Flops:
Macbook air m3
1 core CPU: frequenza massima x 2 (FMA)* x N (SIMD)**
Step 1: FLOPS per core
frequenza ~ 4.06 GHz (valore tipico boost M3) stima presa da internet , l'informazione tecnica non l'ho trovata accessibile.
SIMD ~ 8 (single precision)
N = 2
La CPU Apple M3 supporta istruzioni SIMD ASIMD (NEON) a 128 bit; utilizzando dati in doppia precisione (64 bit) il fattore SIMD risulta N=2.
FMA = 2
FLOPS/core=f×2×SIMD
= 16.25 GFLOPS
I nodi MPI sono implementati tramite container Docker (wn1, wn2) eseguiti sul medesimo MacBook Air M3. Essi rappresentano nodi logici del cluster, ma condividono lo stesso hardware fisico; pertanto le prestazioni teoriche non vengono moltiplicate per il numero di container.
1 nodo CPU: prestazioni di 1 core x numero di core fisici (no hyper-threading)
= Flops(core) * 8 = 129.92 GFLOPS
GPU clock tipico stimato: ~1.3 GHz
1 core GPU: frequenza massima x 2 (FMA) = 2.6 GHz
1 GPU: prestazioni di 1 core x numero di core = 2.6 * 10 = 26 GHz
La GPU Apple M3 non è basata su architettura CUDA e non dispone di CUDA cores. Le prestazioni sono quindi stimate utilizzando il modello semplificato basato su frequenza di clock e numero di core, secondo la formula fornita dal corso. La programmazione GPU avviene tramite API Metal.
